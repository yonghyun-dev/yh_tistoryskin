# REQ-01: 티스토리 모던 개발 블로그 스킨

- 요청일: 2026-04-19
- 상태: 분석 중
- 스프린트: 1~5 (예정)

## 요청 원문 (핵심 발췌)

> 개발 공부·회고·이직 준비 내용을 쌓을 개인 티스토리 블로그용 스킨을 만든다.
> Stripe / Apple / Vercel docs 무드의 깔끔함 + 곳곳에 고급스러운 마이크로 인터랙션이 들어간 "조용한데 세련된" 느낌.
> 나중에 챗봇 서버와 연결할 embed 스니펫이 자연스럽게 들어갈 수 있는 구조로 설계한다.

## 핵심 목표

- 개인 개발 블로그용 **티스토리 2.0 스킨** 제작
- 무드: Stripe · Apple · Vercel docs 스타일의 조용하고 세련된 디자인
- **라이트/다크 모드** 모두 지원 (OS 감지 + 수동 토글, localStorage)
- 미래의 **챗봇 embed 스니펫**을 위한 마운트 지점 · 훅 · 전역 설정 API를 선반영

## 블로그 성격

- 타겟 독자: 비슷한 경로를 걷는 주니어~미드 개발자
- 주 컨텐츠: 기술 튜토리얼, 트러블슈팅, 프로젝트 회고, 이직 준비 기록
- 언어 편향: Python 중심, 다만 언어·기술·CS·커리어까지 폭넓음

## 사용자 시나리오

1. 블로그 주인이 개발 공부·회고·이직 포스트를 축적한다
2. 방문자가 메인에서 **카드 그리드**로 포스트를 탐색한다 (썸네일·제목·발췌·태그·날짜)
3. 방문자가 포스트를 열면 **2단 레이아웃**(본문 + 우측 고정 TOC), **읽기 진행률 바**로 몰입한다
4. 방문자가 **라이트/다크**를 전환한다 (OS 감지 + 수동 토글)
5. 추후 챗봇이 연결되면 우측 하단 **위젯** + 본문 `<!--[chatbot]-->` **마커**에 챗봇 렌더

## 디자인 시스템 스펙

### 컬러
- 라이트: 배경 `#FFFFFF`, 텍스트 `#0A0A0A`
- 다크: 배경 `#0A0E1A`, 텍스트 `#E8E8EC`
- 포인트 그라데이션: `#5B5BF5 → #8B5CF6` (제목 악센트, 링크 hover, 주요 CTA에만 절제해서 사용)

### 타이포
- 본문·제목: **Pretendard** (Regular ~ ExtraBold)
- 코드: **JetBrains Mono**
- line-height 1.7 ~ 1.8, 여백 넉넉

## 기술 제약

- **티스토리 2.0 스킨 구조**: `skin.html`, `index.xml`, `style.css`, `images/`
- **치환자**: `##_xxx_##` 패턴
- **조건부 블록**: `<s_list>`, `<s_article_rep>`, `<s_notice>`, `<s_guest>`, `<s_tag_label>`, `<s_cover_group>`, `<s_permalink_article_rep>` 등
- **서버사이드 코드 불가** — 정적 HTML + CSS + JS만
- **외부 API는 클라이언트 JS**로만 호출
- **모바일 반응형 필수**

## 외부 의존성 (최소화 원칙)

- Pretendard (CDN 예정)
- JetBrains Mono (CDN 예정)
- 코드 하이라이터 1개 (설계 단계에서 결정)

## 유저 제안 스프린트 (참고)

PM 최종 판단은 설계 단계에서 결정. researcher 조사 결과 반영 후 조정 가능.

- Sprint 1: 스킨 기본 구조 + 디자인 시스템 (CSS 변수 토큰, 라이트/다크 전환)
- Sprint 2: 메인 목록 + 프로필 + 카테고리/태그 + 검색
- Sprint 3: 개별글 + TOC + 코드 하이라이팅 + 진행률 바
- Sprint 4: 마이크로 인터랙션 + 관련글/이전-다음 + 반응형 마감
- Sprint 5: 챗봇 스니펫 마운트 훅 + 전역 설정 API + 테스트용 더미 UI

## PM 수임 사항 (유저 당부)

- 티스토리 스킨 구조·치환자·블록 규칙은 **researcher로 공식 가이드 조사 후** 설계에 반영 (추측 금지)
- 디자인 시스템은 **CSS 변수 기반**
- 모든 애니메이션은 **`prefers-reduced-motion` 대응**
- 외부 의존성 **최소화**
- 디자인 세부 결정은 **시니어 판단**으로 합리적 기본값, 중요 판단만 질문

## 비고

- 블로그 주인 프로필(이름·소개·링크 등)은 `index.xml` 변수로 빼서 티스토리 관리자에서 편집 가능하게 할 예정. 스킨 자체에는 placeholder만 둔다.
- 유저가 외부 문서 리서치를 "researcher agent"로 지칭했으나, researcher는 코드 조사 전용(Read/Grep/Glob)이므로 외부 공식 문서 조사는 `general-purpose` agent로 위임한다.
