현재 프로젝트의 전체 상태를 확인하고 다음 행동을 안내한다.

## 확인 순서

아래 문서를 순서대로 읽고 상태를 정리한다:

1. `docs/sprint.md` — 현재 스프린트 상태
2. `docs/requirements.md` — 요구사항 인덱스
3. `docs/agents/*.md` — 각 Agent 작업 현황
4. `docs/design/` — 설계 문서 존재 여부
5. `docs/context/` — 인수인계 문서 존재 여부

## 출력 형식

```markdown
## 현재 상태

### 스프린트
- 번호: [N]
- 목표: [목표]
- 단계: [분석/설계/구현/검증]
- 상태: [진행 중/대기/완료]

### 진행 중인 작업
- [현재 하고 있는 것]

### Agent 현황
| Agent | 상태 | 작업 |
|:--|:--|:--|
| reviewer | [대기/작업 중] | [내용] |
| test-writer | [대기/작업 중] | [내용] |
| researcher | [대기/작업 중] | [내용] |
| doc-writer | [대기/작업 중] | [내용] |
| design-reviewer | [대기/작업 중] | [내용] |

### 파킹랏
- [대기 중인 추가 요청]

### 설계 문서
- data-model.md: [있음/없음]
- api-spec.md: [있음/없음]
- architecture.md: [있음/없음]
- 다이어그램: [N개]

---

## 다음에 해야 할 일

1. [가장 우선순위 높은 작업]
2. [그 다음 작업]
3. [그 다음 작업]

## 추천 명령어

- [상황에 맞는 /skill 또는 Agent 추천]
```

## 상황별 다음 행동 판단 기준

| 현재 상태 | 다음 행동 |
|:--|:--|
| 스프린트 없음 (대기) | "/pm으로 새 작업을 시작하세요" |
| 분석 단계 | "요구사항 확정 후 설계로 넘어가세요" |
| 설계 단계 | "/tech-decision, /data-modeling, /api-design 진행하세요" |
| 구현 단계 | "sprint.md의 작업 목록을 확인하고 구현을 계속하세요" |
| 검증 단계 | "reviewer, test-writer에게 위임하세요" |
| 스프린트 완료 | "파킹랏 정리 후 다음 스프린트를 계획하세요" |
| 파킹랏에 항목 있음 | "파킹랏 항목을 다음 스프린트에 편입할지 판단하세요" |
| Agent 작업 진행 중 | "Agent 결과를 기다리거나 확인하세요" |
| 설계 문서 없음 | "구현 전에 /data-modeling, /api-design으로 설계하세요" |
