# 상태 확인 체크리스트

/status에서 프로젝트 상태를 확인할 때 빠뜨리면 안 되는 항목.

---

## 확인 순서

### 1. 스프린트 상태 (docs/sprint.md)
- [ ] 현재 스프린트 번호
- [ ] 현재 단계 (분석/설계/구현/검증)
- [ ] 진행 중인 작업
- [ ] 완료 기준 달성 여부
- [ ] 파킹랏 항목 존재 여부

### 2. 요구사항 (docs/requirements.md)
- [ ] 진행 중인 요구사항
- [ ] 백로그 항목
- [ ] 완료된 요구사항 수

### 3. PM 작업 (docs/pm/)
- [ ] 최근 PM 작업 파일
- [ ] 미해결 판단 사항

### 4. Agent 현황 (docs/agents/*/index.md)
- [ ] 각 Agent의 현재 작업 상태
- [ ] 대기 중인 작업
- [ ] 최근 완료된 작업

### 5. 설계 문서 (docs/design/)
- [ ] data-model.md 존재 여부
- [ ] api-spec.md 존재 여부
- [ ] tech-decisions.md 존재 여부
- [ ] 다이어그램 존재 여부

### 6. 인수인계 (docs/context/)
- [ ] session_handoff.md 존재 여부
- [ ] 최종 수정 날짜

---

## 다음 행동 판단 트리

```
스프린트 상태?
├── 대기 → "/pm으로 새 작업 시작"
├── 분석 중 → "요구사항 확정 후 설계로"
├── 설계 중 → "/tech-decision → /data-modeling → /api-design"
├── 구현 중 → "sprint.md 작업 목록 확인 후 구현 계속"
├── 검증 중 → "reviewer + test-writer + doc-writer 위임"
└── 완료 → "파킹랏 정리 → 다음 스프린트 계획"

설계 문서?
├── 없음 → "구현 전에 설계 먼저"
└── 있음 → OK

Agent 작업?
├── 대기 중 있음 → "Agent 실행 필요"
├── 진행 중 → "결과 대기"
└── 전부 완료 → OK

파킹랏?
├── 항목 있음 → "다음 스프린트에 편입 검토"
└── 비어있음 → OK
```
