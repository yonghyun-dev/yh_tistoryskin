# 디버깅 및 수정 가이드

## 1. 버그 분류 체계

| 분류 | 설명 | 예시 | 긴급도 |
|:--|:--|:--|:--|
| **크래시** | 앱이 죽거나 500 에러 | 서버 다운, 화이트스크린 | 즉시 |
| **데이터 손상** | 데이터가 잘못 저장/삭제 | 주문 금액 0원, 중복 생성 | 즉시 |
| **보안** | 인증 우회, 데이터 노출 | 다른 사용자 정보 접근 | 즉시 |
| **기능 오작동** | 동작하지만 결과가 틀림 | 필터 안 됨, 정렬 역순 | 높음 |
| **성능** | 느리거나 타임아웃 | 5초 이상 응답 | 보통 |
| **UI 깨짐** | 화면이 이상하게 보임 | 레이아웃 틀어짐, 글자 잘림 | 낮음 |
| **엣지 케이스** | 특정 조건에서만 발생 | 빈 문자열, 한글, 특수문자 | 낮음 |

## 2. 원인 탐색 순서

```
1단계: 에러 메시지 정확히 읽기
  └── 에러 타입, 파일, 라인, 스택 트레이스

2단계: 재현 확인
  └── 어떤 입력으로, 어떤 순서로, 항상/가끔?

3단계: 최근 변경 확인
  └── git log --oneline -10
  └── git diff HEAD~1

4단계: 관련 코드 추적
  └── 에러 발생 지점 → 호출자 → 데이터 소스

5단계: 가설 → 검증
  └── "이게 원인일 것이다" → 로그/디버거로 확인
  └── 한 번에 하나의 변수만 바꿔서 테스트
```

## 3. 수정 영향도 판단 매트릭스

| 수정 대상 | 영향 확인 사항 |
|:--|:--|
| 도메인 모델 필드 변경 | API 응답, DB 스키마, 프론트 타입 전부 확인 |
| API 응답 형식 변경 | 프론트엔드 호출 코드 전부 확인 |
| DB 스키마 변경 | 마이그레이션 필요, 기존 데이터 호환성 |
| 환경변수 추가/변경 | .env.example 업데이트, 배포 설정 |
| 의존성 버전 변경 | breaking changes 확인 |
| 유틸 함수 수정 | 해당 함수를 호출하는 모든 곳 확인 |
| 미들웨어 수정 | 모든 엔드포인트에 영향 |

## 4. 수정 전 체크리스트

- [ ] 에러를 재현할 수 있는가
- [ ] 원인을 정확히 파악했는가 (추측이 아닌 확인)
- [ ] 수정 범위가 최소한인가
- [ ] 수정이 다른 기능을 깨뜨리지 않는가
- [ ] 관련 테스트가 있는가 (있으면 먼저 실행)
- [ ] 수정 후 원래 에러가 해결되었는가

## 5. 수정 후 체크리스트

- [ ] 원래 버그가 해결되었는가
- [ ] 새로운 버그가 생기지 않았는가
- [ ] 관련 테스트가 통과하는가
- [ ] 수정 내용을 유저에게 보고했는가
- [ ] 같은 유형의 버그가 다른 곳에도 있는지 확인했는가
