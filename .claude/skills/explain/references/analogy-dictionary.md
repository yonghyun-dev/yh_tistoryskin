# 비유 사전

/explain에서 어려운 개념을 설명할 때 사용하는 비유 모음.
새 세션에서도 일관된 비유를 사용하기 위해 정리한다.

---

## 아키텍처/구조

| 개념 | 비유 | 설명 |
|:--|:--|:--|
| API | 음식점 메뉴판 | 뭘 주문(요청)할 수 있는지 목록이 정해져 있다 |
| REST API | 우체국 | 편지(요청)에 주소(URL)와 목적(GET/POST)을 적어 보내면 답장(응답)이 온다 |
| 미들웨어 | 건물 입구 보안 게이트 | 모든 사람(요청)이 통과해야 하는 검문소 |
| 라우터 | 안내 데스크 | 요청이 오면 "이건 저쪽으로 가세요"라고 안내하는 역할 |
| 클린 아키텍처 | 양파 껍질 | 바깥 껍질(UI)을 벗겨도 안쪽(비즈니스 로직)은 영향 없다 |
| 의존성 주입 | 레고 블록 | 부품을 교체할 수 있게 끼워넣는 방식 |
| 마이크로서비스 | 전문 식당 거리 | 한 가게가 모든 음식을 하는 대신, 각 가게가 전문 메뉴만 한다 |
| 모노리스 | 백화점 | 한 건물에 모든 게 다 있다 |
| 프록시 | 대리인 | 내가 직접 가는 대신 대리인이 대신 가서 전해준다 |
| 로드밸런서 | 은행 번호표 시스템 | 여러 창구(서버)에 고객(요청)을 골고루 분배 |
| CDN | 편의점 체인 | 본사(원본 서버) 대신 가까운 편의점(엣지 서버)에서 바로 받는다 |

## 데이터

| 개념 | 비유 | 설명 |
|:--|:--|:--|
| 데이터베이스 | 엑셀 파일 | 데이터를 표(테이블)로 정리해서 저장하는 곳 |
| 인덱스 | 책 뒤의 찾아보기 | 전체를 다 읽지 않고 원하는 내용을 빠르게 찾는다 |
| 정규화 | 주소록 정리 | 같은 정보를 여러 곳에 적지 않고 한 곳에만 적는다 |
| 캐시 | 책상 위 메모지 | 자주 보는 정보를 가까이 두고 빠르게 확인한다 |
| 트랜잭션 | 은행 이체 | "보내기"와 "받기"가 둘 다 성공하거나 둘 다 취소된다 |
| ORM | 통역사 | 코드(한국어)와 DB(SQL)의 언어를 통역해준다 |
| 마이그레이션 | 이사 | 데이터 구조를 옛집에서 새집으로 옮기는 것 |
| N+1 쿼리 | 물건을 하나씩 사러 가는 것 | 한 번에 목록으로 사면 되는데, 하나씩 반복해서 가게에 간다 |
| JOIN | 두 개 표 합치기 | 공통 열(학번, 주문번호)을 기준으로 두 표를 합친다 |
| 벡터 DB | 비슷한 그림 찾기 | "이것과 비슷한 것" 을 찾아준다 (의미 기반 검색) |

## 인증/보안

| 개념 | 비유 | 설명 |
|:--|:--|:--|
| 인증 (Authentication) | 신분증 확인 | "당신이 누구인지" 확인하는 것 |
| 인가 (Authorization) | 출입 권한 확인 | "당신이 여기에 들어갈 수 있는지" 확인하는 것 |
| JWT | 놀이공원 팔찌 | 한번 입장(로그인)하면 팔찌(토큰)로 놀이기구를 탈 수 있다 |
| OAuth | 대리 출입증 | "카카오가 이 사람 확인해줬어요"라고 대신 인증해주는 것 |
| 해싱 | 고기 분쇄기 | 원본(비밀번호)을 넣으면 결과(해시)가 나오지만, 결과로 원본을 복원할 수 없다 |
| CORS | 국경 검문소 | "이 나라(도메인)에서 온 요청은 허용/차단" |
| Rate Limiting | 뷔페 접시 제한 | 한 번에 너무 많이 가져가지 못하게 횟수를 제한 |
| 환경변수 | 금고에 넣은 비밀번호 | 코드에 직접 안 쓰고 따로 안전하게 보관 |

## 프론트엔드

| 개념 | 비유 | 설명 |
|:--|:--|:--|
| 컴포넌트 | 레고 블록 | 작은 블록(버튼, 카드)을 조합해서 큰 화면을 만든다 |
| 상태 (State) | 전광판 | 값이 바뀌면 화면도 자동으로 바뀐다 |
| Props | 택배 상자 | 부모가 자식에게 데이터를 전달하는 상자 |
| Hook | 도구 상자 | 필요한 기능(상태관리, API호출)을 꺼내 쓴다 |
| 렌더링 | 그림 다시 그리기 | 데이터가 바뀌면 화면을 다시 그린다 |
| 가상 DOM | 설계도 비교 | 진짜 집(DOM)을 고치기 전에 설계도(가상 DOM)에서 뭐가 바뀌었는지 먼저 비교한다 |
| SSR/SSG | 미리 만든 도시락 vs 주문 즉시 조리 | SSG는 미리 만들어놓고, SSR은 주문 올 때마다 만든다 |

## AI/ML

| 개념 | 비유 | 설명 |
|:--|:--|:--|
| RAG | 오픈북 시험 | 시험(질문)을 볼 때 교과서(문서)를 찾아보고 답한다 |
| Fine-tuning | 전문 과외 | 일반 선생님(범용 모델)에게 특정 과목을 집중 교육하는 것 |
| Embedding | 좌표로 변환 | 글자를 숫자(좌표)로 바꿔서 "가까운 것 = 비슷한 것" 으로 만든다 |
| 프롬프트 | 주문서 | AI에게 "이런 식으로 해줘"라고 상세히 적는 것 |
| 토큰 | 글자 조각 | AI가 글을 읽는 최소 단위 (대략 한국어 1글자 ≈ 2~3토큰) |
| Temperature | 창의성 다이얼 | 낮으면 안정적, 높으면 다양한(때로는 이상한) 답이 나온다 |
| 온톨로지 | 백과사전 분류 체계 | "강아지는 동물이고, 동물은 생물이다" 같은 지식 구조 |
| 에이전트 | 비서 | 알아서 도구를 사용하고, 여러 단계를 거쳐 일을 완료하는 AI |

## DevOps/인프라

| 개념 | 비유 | 설명 |
|:--|:--|:--|
| Docker | 이사 박스 | 앱과 필요한 모든 것을 박스에 넣어서 어디서든 똑같이 실행 |
| CI/CD | 자동 공장 라인 | 코드를 넣으면 자동으로 테스트 → 빌드 → 배포 |
| 환경 (dev/staging/prod) | 연습실/리허설/본공연 | 개발 → 테스트 → 실제 서비스 단계 |
| 스케일링 | 가게 확장 | 손님이 많으면 직원(서버)을 더 고용한다 |
| 모니터링 | 건강 검진 | 서버가 아프기(장애) 전에 미리 상태를 체크한다 |
